# Bitácora — 2026-07-09

## Objetivo del día
Arrancar desde cero el sistema de designación de materias a docentes: instalar Laravel,
definir el modelo de datos, dejar la primera sección (Designación de Docentes) funcionando
con datos de prueba, y dejar el repositorio listo para trabajar en equipo.

## Qué se hizo
- Se instaló un proyecto Laravel nuevo, configurado con SQLite como base de datos de
  desarrollo (no tenemos acceso todavía a la base de datos real de la universidad).
- Se creó `CLAUDE.md` con el contexto del proyecto y las reglas de trabajo (documentación,
  commits, qué no hacer sin preguntar).
- Se creó `docs/ERD.md` con el diagrama del modelo de datos y notas de las reglas del negocio
  que ya se conocen (materia en varias mallas, grupos reutilizables, gestión/periodo como
  catálogos globales, etc.).
- Se crearon las 9 tablas del modelo con sus migraciones: `carreras`, `materias`,
  `malla_curricular`, `grupos`, `docentes`, `gestiones`, `periodos`, `designaciones` y
  `designaciones_historial`, junto con sus modelos Eloquent y relaciones.
- Se construyó la primera y única sección del sistema por ahora: "Designación de Docentes",
  con listado, formulario de creación/edición y vista de historial por designación.
- El historial se registra automáticamente: cada vez que se edita una designación (docente,
  materia, grupo, gestión, periodo o estado), queda guardado qué cambió y cuándo.
- Se cargaron datos de prueba realistas: 4 carreras (Informática, Matemáticas, Medicina,
  Civil), 9 materias con sigla real (ej. "INF-520"), grupos, 8 docentes con nombres
  bolivianos, 2 gestiones, 3 periodos y 5 designaciones de ejemplo.
- Se probó el flujo completo (listado, crear, editar, ver historial) levantando el servidor
  local y verificando que cada página respondiera correctamente antes de dar el trabajo
  por terminado.
- Se preparó el repositorio para GitHub: `.gitignore` revisado para que no suba `.env`,
  `vendor/`, `node_modules/` ni el archivo `database/database.sqlite`. Se hizo el primer
  commit con todo el bootstrap.

## Decisiones tomadas
- El template de interfaz Color Admin v2 (SeanTheme) todavía no está disponible en este
  equipo. En vez de inventar un HTML parecido o descargarlo de otro lado (es un template de
  pago), se construyeron las vistas con Bootstrap 5 simple, dejando anotado en `CLAUDE.md`
  y `README.md` que hay que reemplazarlas cuando lleguen los archivos reales del template.
  Esto se decidió después de confirmarlo directamente antes de avanzar.
- Los nombres de tabla en español (`gestiones`, `designaciones`, `malla_curricular`,
  `designaciones_historial`) no coinciden con lo que Laravel adivina automáticamente a
  partir del nombre del modelo (el pluralizador de Laravel está pensado para inglés). Se
  corrigió indicando el nombre de tabla explícitamente en cada modelo donde hacía falta.
- Por la misma razón, la ruta de edición de una designación tampoco adivinaba bien el
  nombre del parámetro (`designacione` en vez de `designacion`). Se corrigió indicándolo
  explícitamente en `routes/web.php`.
- El motor de PHP de este equipo (XAMPP) no tenía habilitada la extensión de SQLite. Se
  habilitó en `php.ini` porque sin eso el proyecto no puede correr en modo desarrollo.
- Al editar una designación, se decidió registrar automáticamente en el historial cualquier
  cambio de docente, materia, grupo, gestión, periodo o estado — es justamente para lo que
  existe la tabla `designaciones_historial`, y no depende de ninguna regla de negocio sin
  confirmar todavía.
- La advertencia de "menos de 6 horas de carga total" que pedía el flujo original **no se
  implementó**: no manejamos horas por materia en los datos disponibles. Queda pendiente,
  ver "Bloqueos / dudas".

- No hay información de horas de carga por materia, así que no se puede implementar la
  advertencia de "menos de 6 horas de carga total" para un docente. Habría que definir de
  dónde sale esa información antes de construir esa validación.
- El sistema de roles lo va a implementar otro equipo — por ahora cualquiera que entre al
  sistema puede crear, editar o eliminar designaciones sin restricción.
