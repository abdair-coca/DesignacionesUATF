## Decisiones tomadas
- Los tests automatizados corren contra la base de datos real de desarrollo
  (`designaciones_uatf`, Postgres) en vez de una base de datos en memoria (SQLite), que es
  lo que trae Laravel por defecto. Motivo: dos partes del sistema (el borrado bloqueado de
  Fase 2 y el buscador de docentes) usan comandos específicos de Postgres que SQLite no
  entiende de la misma forma — correr los tests contra SQLite directamente tiraba errores
  que no tienen nada que ver con bugs reales del sistema. Para que esto sea seguro, cada
  test corre dentro de una transacción que se deshace automáticamente al terminar (no deja
  basura ni modifica los datos de prueba ya sembrados).
- No se pudo crear una base de datos aparte solo para tests (hubiera sido más prolijo)
  porque el usuario configurado en `.env` no tiene permiso para crear bases de datos
  nuevas en Postgres. Ver "Bloqueos / dudas" abajo.
- Se agregaron "factories" (generadores de datos de prueba) para las 6 entidades de
  catálogo y para designaciones, que no existían todavía (solo existía la de usuarios).

## Cómo probar (end-to-end, en el navegador)

1. Levantar el sistema: `composer run dev` (o en dos terminales, `php artisan serve` y
   `npm run dev`).
2. Entrar a `http://127.0.0.1:8000/login` y loguearse con una de las cuentas del seeder
   (`abdair@uatf.edu.bo` / `password` o `supervisor@uatf.edu.bo` / `password`).
3. Confirmar que después de loguearse cae directo en el Dashboard (grupos sin cubrir,
   conteo por estado, docentes bajo el límite de horas).
4. Recorrer cada pantalla de "Catálogos" en el menú lateral (Carreras, Materias, Grupos,
   Docentes, Gestiones, Periodos): crear un registro nuevo, editarlo, y borrarlo. En
   Carreras/Materias/Grupos, probar también que el sistema avisa (no deja borrar) si el
   registro tiene datos dependientes — por ejemplo, intentar borrar una carrera que ya
   tiene materias cargadas.
5. Ir a "Designación de Docentes": probar crear una designación individual, la asignación
   masiva por carrera, y copiar designaciones de una gestión a otra.
6. Al crear/editar una designación, cambiar el docente/materia/grupo y verificar que
   aparece el aviso de horas (si supera 6h) o de choque (si el grupo ya tiene otro
   docente) sin recargar la página — y que aun así se puede guardar (es solo aviso).
7. Entrar al listado de designaciones y probar los filtros por carrera, gestión, periodo
   y estado.
8. Cerrar sesión y confirmar que cualquier URL del sistema (por ejemplo `/dashboard` o
   `/carreras`) redirige de nuevo a `/login` en vez de mostrar la pantalla.
9. Correr la suite automatizada para confirmar que todo lo anterior también está cubierto
   por tests: `php artisan test` (debería mostrar 41 tests en verde).
