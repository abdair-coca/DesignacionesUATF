### Segunda parte — Formulario cascada

- **Backend — `catalogos()` enriquecido**: Ahora incluye `carreras` (todas las carreras activas) y enriquece cada docente con `historial_materias` (array de IDs de materias que dictó anteriormente). Se usa una query SQL con `distinct` para extraer pares docente-materia de la tabla `designaciones`.
- **Frontend — `FormFields.jsx` reescrito con modo cascading**: Cuando se pasa `cascading={true}` (modo Create), los selects se encadenan: Carrera → Materia → Grupo → Docente. Cada selección filtra las opciones del siguiente select. Si no se pasa `cascading` (modo Edit), mantiene el layout plano original.
- **Filtros cascada en client-side**: Se usan `useMemo` para filtrar materias por carrera_id, grupos por materia_id, y docentes por historial/carrera. Al cambiar carrera se resetea materia, grupo y docente. Al cambiar materia se resetea grupo y docente.
- **Ordenamiento inteligente de docentes**: Cuando se selecciona una materia, los docentes se reordenan en 3 categorías con `<optgroup>`: (1) docentes que dictaron esa materia antes, (2) docentes de la misma carrera, (3) todos los demás. Esto facilita elegir al docente más adecuado.
- **`Create.jsx` actualizado**: Agrega `Id_carrera` al `useForm()`, pasa `carreras` y `cascading` a FormFields.

## Decisiones tomadas
- **`gestionActual` = `Gestion::max('id')`**: Se usa el ID más alto como "gestión más reciente". Es una aproximación razonable ya que los IDs son incrementales y la universidad crea una gestión nueva por año.
- **`periodoActual` = `Periodo::min('id')`**: Se usa el primer período como default. El usuario puede cambiarlo fácilmente en el form.
- **Forzar `estado` en `withValidator` en vez de en el controller**: Se hizo en el FormRequest para mantener la lógica de validación/transformación encapsulada. El controller simplemente hace `$request->validated()` y obtiene el estado ya forzado.
- **`showEstado` en FormFields**: Se mantuvo el campo de estado solo en Edit para preservar la funcionalidad existente mientras se simplifica Create.
- **Formulario cascada en client-side**: Se optó por filtrar todo en el navegador (useMemo) en vez de hacer requests AJAX al servidor. Es más rápido para el usuario (sin latencia de red) y las tablas de catálogos son pequeñas (< 100 registros cada una). Se podrían cargar más de 200 docentes, pero con el filtro de horas ya quedan pocos.
- **`Id_carrera` como campo no persistido**: La carrera es un campo de filtrado en el frontend, no se envía al servidor. No se agregó a la migración ni al modelo. Es un campo auxiliar del formulario.
- **Docentes enriquecidos con `historial_materias`**: Se decidió enviar el historial completo de materias por docente en la carga inicial (una sola query SQL) en vez de hacer requests separados por materia. Es más eficiente y permite el filtrado instantáneo en el frontend.

## Sesión 2 — Sistema de copiar/pegar designaciones

### Objetivo
Implementar un sistema de copiar/pegar designaciones entre carreras/periodos usando el portapapeles del navegador (localStorage + UI flotante), reemplazando la página dedicada de "Copiar designaciones" que existía antes.

### Qué se hizo

#### Etapa 1: Selección multi-fila (Ctrl+Click)
- **`useSelection.js`**: Hook personalizado que maneja un `Set` de IDs seleccionados. Soporta Ctrl+Click (toggle individual), toggleAll, clearAll. El estado es un `Set` de React (inmutable). Retornado como `useMemo` para evitar re-renders innecesarios.
- **Carrera.jsx**: Se agregó `onClick` handler en las filas de ambas tabs (roster y resumen) que llama a `seleccion.toggle(id, ctrlKey)`. Las filas seleccionadas tienen fondo azul claro con ring inset.
- **Escape**: Limpia la selección completa.

#### Etapa 2: Copiar al portapapeles (localStorage)
- **`useClipboard.js`**: Funciones puras (sin React): `clipboardCopy(filas, origen)`, `clipboardRead()`, `clipboardClear()`. Almacenan JSON en `localStorage` con key `designaciones_clipboard` y expiración de 30 minutos. Emite evento custom `clipboard-changed` para que el banner se actualice.
- **Carrera.jsx**: Botón "Copiar (N)" que aparece solo cuando hay filas seleccionadas. Handler `copiarSeleccionadas` construye el payload con info de materia, docente, grupo, carrera, gestión y periodo origen. Ctrl+C en el teclado también dispara la copia (sobresee el comportamiento del navegador cuando hay selección en la tabla).
- **ClipboardBanner.jsx**: Componente flotante fijo en el bottom-center que muestra cuántas designaciones están copiadas y de qué origen. Polling simple (escucha eventos `clipboard-changed` y `storage`). Sin Context React (ver decisión abajo). Botón "Limpiar" para descartar.

#### Etapa 3: Pegar designaciones (frontend + backend)
- **PasteModal.jsx**: Modal que muestra las filas del clipboard con checkbox de selección individual/todas. Advierte si las filas provienen de otra carrera. Muestra destino (gestión/periodo actual). Botón "Pegar (N)" que hace POST al backend.
- **Backend `pegar()`**: Endpoint `POST /designaciones/pegar` (JSON). Valida cada fila (`Id_docente`, `Id_materia`, `Id_grupo`). Usa transacción. Salta grupos que ya tienen designación activa en destino. Salta docentes que excederían el límite de horas. Devuelve `{creadas, saltadas, creadas_ids}`.
- **Backend `deshacerPegado()`**: Endpoint `POST /designaciones/deshacer-pegado`. Elimina designaciones por ID solo si están en estado "propuesta". Protege contra eliminación de designaciones aprobadas.
- **Undo**: El modal muestra resultado con botón "Deshacer" que llama a `deshacerPegado` y recarga la página.

#### Limpieza
- **Eliminado `Copiar.jsx`**: La página dedicada de copiar ya no es necesaria con el nuevo flujo de clipboard.
- **Eliminado `ClipboardContext.jsx`**: Causaba crash TDZ (Temporal Dead Zone) al ser importado desde un page component. Se reemplazó por polling directo de localStorage en el Banner.
- **Eliminado rutas `GET/POST /designaciones/copiar`**: Ya no existen.
- **Eliminado `DesignacionMasivaController` legacy**: Se mantuvo solo `pegar()` y `deshacerPegado()`. Se eliminaron `copiarForm()`, `copiarStore()`, `crearEnLote()`, `copiarTodas()`.
- **Test actualizado**: Se eliminó la referencia a `designaciones.copiar` del `GuestRedirectTest` data provider. 58/58 tests pasan.

### Decisiones tomadas
- **Clipboard en localStorage, no en React state**: Se usó `localStorage` como almacenamiento del clipboard porque: (a) persiste entre navegación de páginas Inertia, (b) se sincroniza entre pestañas via evento `storage`, (c) evita la necesidad de un Provider de React Context que causaba problemas de TDZ con el bundler.
- **Sin Context React**: El `ClipboardContext` causaba un crash `Cannot access 'ee' before initialization` en producción. Se determinó que el bundler (Rolldown/Vite 8) tiene un bug con ciertos patrones de imports circulares entre Context y page components. La solución fue usar funciones puras + polling, eliminando la dependencia de React Context.
- **Axios descartado — causaba TDZ también**: Se intentó instalar `axios` para manejar CSRF automáticamente, pero su import corruptó el orden de inicialización de módulos en el bundler Rolldown (Vite 8). Se eliminó y se usó `fetch` nativo con XSRF-TOKEN cookie + meta tag CSRF.
- **`pegar()` retorna JSON, no redirect**: A diferencia de otros endpoints Inertia, `pegar()` retorna JSON porque el frontend necesita los `creadas_ids` para el undo, y el reload de la página se hace desde JavaScript con `router.reload()`.
- **Solo deshacer propuestas**: `deshacerPegado()` solo elimina designaciones en estado "propuesta" para no romper designaciones que ya hayan sido aprobadas.

### Error TDZ no resuelto (Temporal Dead Zone)

**Descripción del problema:**
Al cargar cualquier página del sistema, el bundle JavaScript de producción lanza `Uncaught ReferenceError: Cannot access 'XX' before initialization` (donde `XX` es un nombre minificado como `ee`, `ne`, `te`). El error ocurre durante la inicialización de módulos React, no durante el render. La página queda en blanco.

**Qué se intentó (cronológicamente):**
1. Importar `useSelection` + `useClipboard` en Carrera.jsx → crash
2. Separar funciones puras (`Hooks/useClipboard.js`) del React Context (`Components/ClipboardContext.jsx`) → crash persiste
3. Revertir todo a estado sin clipboard → funciona
4. Re-agregar solo `useSelection` (sin clipboard) → funciona
5. Re-agregar `clipboardCopy` (función pura, no context) + ClipboardBanner en app.jsx → funciona
6. Agregar PasteModal + `axios` → crash en TODAS las páginas
7. Reemplazar `axios` por `fetch` nativo → crash persiste

**Patrón identificado:**
- El crash NO es por React Context (ClipboardContext eliminado, Banner usa polling)
- El crash NO es por la función pura `clipboardCopy` (funciona sin PasteModal)
- El crash SÍ se activa al importar `PasteModal` en Carrera.jsx (que a su vez importa `axios` o cualquier módulo externo pesado)
- El crash se manifiesta en TODAS las páginas porque `import.meta.glob` con `eager: true` carga todos los componentes de página al inicio
- El bundler Vite 8 / Rolldown parece tener un bug con el orden de inicialización cuando se combinan imports estáticos pesados en page components

**Estado actual del clipboard (lo que SÍ funciona):**
- ✅ Ctrl+Click para seleccionar filas en la tabla de carrera
- ✅ Botón "Copiar (N)" + Ctrl+C para copiar al localStorage
- ✅ Banner flotante que muestra designaciones copiadas
- ✅ Ruta `POST /designaciones/pegar` (backend) funcional
- ✅ Ruta `POST /designaciones/deshacer-pegado` (backend) funcional
- ❌ PasteModal NO se puede importar en Carrera.jsx sin causar TDZ

**Lo que NO se pudo completar:**
- El modal de pegado (PasteModal.jsx) no se puede integrar en la vista de carrera por el bug de TDZ del bundler
- El flujo completo copiar→pegar queda incompleto en el frontend (el backend está listo)
- El undo/deshacer queda inaccesible desde la interfaz

**Posibles soluciones a futuro (requieren investigación):**
1. Actualizar Vite/Rolldown a una versión que corrija el bug de inicialización
2. Convertir PasteModal a lazy import (`React.lazy()`) para que no se cargue en el bundle principal
3. Extraer PasteModal a una ruta/página independiente en vez de un modal dentro de Carrera
4. Migrar a otro bundler (esbuild, Webpack) que no tenga este bug
5. Investigar si el bug está en la forma en que Rolldown tree-shakea módulos con imports pesados
